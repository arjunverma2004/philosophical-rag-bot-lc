Philosophical RAG Assistant 📚A Retrieval-Augmented Generation (RAG) application built with Streamlit, LangChain, and Google Gemini. This tool allows users to upload PDF books, index them into a local vector database, and ask complex philosophical questions. The AI is strictly prompted to answer only based on the provided text, ensuring high fidelity to the original authors' meanings without hallucinating outside knowledge.✨ Core Features Explained1. 📚 Multi-Document Processing & UploadHow it works: Users can select and upload multiple PDF books simultaneously through the Streamlit sidebar. The files are securely saved to a local uploaded_books directory.Benefit: Allows you to easily build a massive, customized local library of philosophical texts without having to run terminal scripts manually for each new book.2. ⚡ Smart Caching (Skip Duplicates)How it works: Before chunking and embedding a document, the system checks if the file already exists in the local directory. If it does, it skips the processing pipeline for that specific file.Benefit: Saves significant time and compute resources (and API limits) by ensuring you never waste time re-embedding a book that is already in your database.3. 🎯 Advanced Retrieval with MMRHow it works: Uses Maximal Marginal Relevance (MMR) instead of standard similarity search when fetching documents from ChromaDB.Benefit: Standard search often returns 3 or 4 chunks of text that say the exact same thing. MMR balances relevance (how well it matches your question) with diversity (ensuring the chunks cover different parts of the text), leading to much richer, well-rounded answers.4. 🔍 Context Transparency (Source Verification)How it works: The web UI includes an expandable "Sources" dropdown that displays the exact paragraphs and chunks retrieved from the vector database.Benefit: Eliminates the "black box" feeling of AI. You don't have to blindly trust the LLM; you can instantly read the exact source text it used to formulate its response and verify its accuracy.5. 💾 Automated Session LoggingHow it works: Every time an answer is generated, the app automatically creates a timestamped .txt file in the saved_responses/ folder. This file contains the date, your original question, the AI's full answer, and the exact text context used.Benefit: Creates a permanent, easily readable history of your philosophical inquiries and research notes for future reference.6. 🧠 Strict "No Hallucination" GuardrailsHow it works: The LangChain prompt template (chains.py) enforces strict behavioral rules on the Gemini model. It is instructed to answer only using the provided text, break down arguments step-by-step, and explicitly state "The text does not provide a clear answer" if the information is missing.Benefit: Ensures high fidelity to the original texts, preventing the AI from bringing in outside knowledge, modern biases, or internet-based interpretations.🛠️ Tech StackFrontend: StreamlitFramework: LangChainVector Database: ChromaDBEmbeddings: HuggingFace (all-MiniLM-L6-v2)LLM: Google Generative AI (gemini-2.5-flash-lite)Document Parsing: PyPDF🚀 InstallationClone the repository:git clone <your-repo-url>
